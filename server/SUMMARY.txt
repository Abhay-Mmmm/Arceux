✅ 1. SYNTHETIC LOG GENERATOR (log_generator.py)
   └─ Emits realistic security events every 1-3 seconds
   └─ 5 event types with weighted distribution
   └─ HTTP POST to ingestion API

✅ 2. DATA MODELS (models.py)
   └─ Pydantic schemas for type safety
   └─ SecurityLog, DetectionSignal, Alert, MetricsSummary
   └─ Enums for event types, severity, signal types

✅ 3. STORAGE LAYER (storage.py)
   └─ Thread-safe in-memory storage
   └─ JSON persistence for alerts
   └─ Automatic memory management
   └─ Clean separation: logs, signals, alerts

✅ 4. DETECTION ENGINE (detection_engine.py)
   └─ 3 sophisticated detection rules:
      • Brute Force: >5 failed logins in 2 minutes
      • Suspicious Login: Geographic anomalies
      • Insider Threat: Privilege escalation → data access
   └─ Stateful pattern matching
   └─ Sliding time windows

✅ 5. CREWAI AGENT SYSTEM (agents/crew_system.py)
   └─ 5 specialized AI agents:
      1. Ingestion Analyst    → Normalizes signals
      2. Threat Analyst       → Classifies threats
      3. Context Enrichment   → Adds behavioral context
      4. Explanation Agent    → Writes human explanations
      5. SOC Manager          → Final prioritization
   └─ Sequential workflow with shared memory
   └─ Graceful fallback on failure

✅ 6. FASTAPI APPLICATION (api.py)
   └─ Log ingestion: POST /logs
   └─ Frontend API:
      • GET /alerts (with filtering)
      • GET /alerts/{id}
      • GET /metrics
      • GET /health
   └─ Background signal processing
   └─ Debug endpoints for POC
   └─ CORS enabled

   └─ requirements.txt      → All dependencies
   └─ .env.example          → Configuration template
   └─ .gitignore            → Version control
   └─ start.sh              → API launcher
   └─ start_all.sh          → Full system launcher
